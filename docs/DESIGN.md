# Playscript 设计存档

从**分镜视频帧**提取台本（镜号 / 角色 / 台词）的工具。

> 注：项目文件夹计划改名为首字母大写 `Playscript`。因当前会话工作目录占用该文件夹，
> Windows 不允许重命名「正在被进程用作当前目录」的文件夹，需**关闭会话后**在资源管理器/终端
> 里改名（Windows 大小写不敏感，改名后路径照常工作，零影响）。

## 背景与约束
- 分镜视频字幕由分镜师用别的软件（TBS 等）制作，**拿不到源工程，只有成片**。
- 字幕**烧录进画面像素**；OCR 是主路径（STT 放弃：分镜片多为草轨/无声，画面已含全部文字）。
- 镜号由使用者**重新排序**，且不重新导出视频 → 需要人工在 PR 时间线上逐帧定位抓取。
  因此最终形态是 **PR 扩展面板**（CEP），而非独立批处理工具。

## 帧画面布局（实测 1920x1080）
| 区域 | 位置 | 内容 | 用途 |
|------|------|------|------|
| 底部中间 | y≈946，居中 | `说话人：台词`（冒号前可含 `&` 表多人） | ✅ 角色 + 台词 |
| 左上 | y≈68 | `ep26_SC002`（使用者重排过的正确镜号） | ✅ 镜号 |
| 右上 | y≈113 / y≈72 | `Sc_002`（分镜师原编号）+ 计数/时间码 | ❌ 丢弃 |
| 顶部 | y≈4 | `说明：…` 场景注释 | ❌ 丢弃 |

**位置不写死**：镜号靠 `SC\d+` 正则 + 左上方位；台词靠「中下部 + 含冒号 + 非『说明』」。容忍上下漂移。

## 解析规则
- **镜号**：左上 `ep26_SCxxx` → 取 SC 后数字（保留前导零）。仅在 SC 上下文把 `一` 当分隔符，绝不全局替换（台词里「一」是常用字）。
- **角色 / 台词**：取中下部（排除「说明」）**同一行的所有文本框**，按 x 拼接成整句，再按冒号（含变体 `：: · ∶`）拆分。
  - 台词常被 OCR 拆成多个框（`大个孑：` + `叩咀阿叩咀。`）→ 必须拼接，否则丢台词。
  - 续行可能**无说话人前缀**（只有台词）→ 不强制要求冒号，整段收作台词，说话人待继承。
- **帧排序**：按 `(集, 镜号, 子序)`，基名(无后缀)子序记 0，排在 `-2/-3` 之前，保证同镜号时间顺序正确
  （文件名直接排序会因 `-`<`.` 把 `sc059` 排到 `sc059-2` 之后）。
- **说话人继承**：续行(无说话人)从**同镜号的上一帧**继承角色（带告警）。
- **合并**：相邻行**同镜号且同角色** → 台词拼接不换行；中间换了别人说 → 另起一行
  （因相邻上一条即那个别人，条件自然不满足）。
- **多说话人拆分**：角色含 `&` → 每人一行、台词复制（合并之后再拆）。

## OCR 准确率三层防线
1. **放大后识别**：OCR 前用 WinRT `BitmapTransform` 放大（默认 **3x**，插值 **Cubic**），
   显著减少「左右结构字被拆成偏旁」的错误。无需 ffmpeg，PR 面板可复用。
2. **偏旁纠错表**（`src/parse/ocrfix.js`）：`亻壬→任`、`氵吉→洁`、`亻十→什` 等精确替换，可随用随补。
3. **人工复核**（M2）：低置信度/异常行标出，快速改错；新纠错可回写纠错表。

> 实测 4 帧样本：Cubic 3x + 纠错表 → 100% 正确，冷启动 ~640ms/帧（<1s）。

## 目录结构
```
src/
  ocr/win_ocr.ps1   Windows.Media.Ocr，输出行文本+坐标(JSON)，支持 -Scale
  ocr/ocr.js        引擎封装（win 本地默认 + cloud 预留），默认 scale=3
  parse/parse.js    整帧行 → 镜号/角色/台词（按特征+方位归类）
  parse/ocrfix.js   偏旁纠错表
  merge/merge.js    相邻同镜号同角色合并
  docx/docx.js      三列表格 docx（对齐 example，可选）
  output/text.js    纯文本输出（默认）：镜号<空格>角色<空格>台词 + 换行
  cli.js            pics 目录 → 识别 → 解析 → 合并 → 输出 + 验证报告
```

## 输出格式
- **默认纯文本**（不生成表格）：`镜号  角色  台词`，两空格分隔，逐行换行。
- 按 `-o` 扩展名选择：`.txt` → 文本；`.docx` → 三列表格（保留）。

## 面板可勾选项（M3 接入 PR 面板的 toggle）
这些当前是 CLI 参数，M3 做成面板复选框，随时可勾：
- 镜号续行：**每行重复** ↔ **留空对齐**（`--merge-shot`）
- 多说话人拆分：`大个子&雪王` → 每人一行、台词复制（`--no-split-speakers` 关闭，默认开）
- 列分隔符（`--sep`，默认两空格）
- 放大倍数 / 插值（`--scale`）
- 合并连接符（`--joiner`，默认直接相连）
- OCR 引擎（本地 / 云端，`--engine`）
- 输出格式（txt / docx）

## PR 抓帧（M3 技术要点）
- QE DOM：`app.enableQE(); qe.project.getActiveSequence().exportFramePNG(CTI.time, path)` —— 一键、非官方 API。
- 兜底：原生「导出帧」(Ctrl+Shift+E) 存入监听文件夹，面板监听到新图即 OCR。两条路共用同一管线。
- **缓存图一律临时 + 用完即删**（存临时目录 → OCR → 立即删除），不提供「保留抓帧」选项。
- **不需手动命名/选路径**：QE DOM 路径由面板自动存放，缓存名是一次性的（计数/时间戳）。
- **镜号来自画面 OCR 标签**（你重排过的号），**行序 = 点击「抓取」的先后**（面板按点击顺序追加行）。
  → 现在靠手动文件名编码「镜号 + 子序」的做法在面板里退休；调序/改镜号在表格内直接完成。
